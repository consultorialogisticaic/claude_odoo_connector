You have been invoked as the Odoo source-installation expert.
The conda environment for the target version is already activated and the Odoo community
repo is cloned at `odoo-workspace/<version>/odoo/`. Your job is to install Python
dependencies and verify the environment is ready to run `odoo-bin`.

---

## Step 1 — Determine the version

Ask or detect which Odoo version is being installed (14.0 through 19.0).
All instructions below are keyed by major version.

---

## Step 2 — Python version gate

| Odoo version | Minimum Python |
|---|---|
| 14.0 | 3.7 |
| 15.0 | 3.7 |
| 16.0 | 3.7 (3.10+ recommended) |
| 17.0 | **3.10** (hard minimum) |
| 18.0 | **3.10** |
| 19.0 | **3.10** |

Verify the active Python satisfies the minimum:
```bash
python --version
```
If the version is too low, stop and tell the user to create a conda env with the right
Python before continuing.

## Step 3 — System-level build dependencies (macOS)

These are needed to compile C-extension wheels (`python-ldap`, `psycopg2`, `lxml`, etc.).

```bash
# Xcode command-line tools (one-time)
xcode-select --install 2>/dev/null || true

# Homebrew packages needed by pip wheels
brew install libxml2 libxslt openldap postgresql
```

> On Linux (Debian/Ubuntu) the equivalent is:
> ```bash
> sudo apt install python3-dev libldap2-dev libpq-dev libsasl2-dev \
>   libxml2-dev libxslt1-dev zlib1g-dev libffi-dev
> ```

---

## Step 4 — pip install

The requirements file is always at the repo root:

```bash
ODOO_SRC="odoo-workspace/<version>/odoo"

# Ensure build tooling is current (critical for v14/v15, harmless for newer)
pip install --upgrade pip setuptools wheel

# Install Odoo's pinned dependencies
pip install -r "$ODOO_SRC/requirements.txt"
```

### Version-specific pip extras

**v14 only** — needs `Mako` (removed in v15):
```bash
pip install Mako
```

**v14 / v15** — older `python-ldap` pin needs explicit build flags on macOS:
```bash
LDFLAGS="-L$(brew --prefix openldap)/lib" \
CPPFLAGS="-I$(brew --prefix openldap)/include" \
pip install python-ldap==3.1.0   # v14
# or python-ldap==3.4.0          # v15/v16
```

**v17 / v18 / v19** — `geoip2` is a new required dep (included in requirements.txt).
No extra action needed, but at runtime the GeoLite2 databases are needed for IP
geolocation. This is optional for demos — safe to ignore.

**v18 / v19** — new deps: `asn1crypto`, `cbor2`, `openpyxl` (all in requirements.txt).

**v19** — requirements.txt has conditional pins per Python version bracket:
- Python < 3.11: one set of pins
- Python 3.11–3.12: another set
- Python 3.12+: another (includes `lxml-html-clean`)
- Python 3.13+: bumps `gevent`, `greenlet`, `psycopg2`, adds `PyPDF` (replaces `PyPDF2`)

pip resolves these automatically. No manual intervention.

## Step 5 — Verify psycopg2

`psycopg2` compiles against `libpq`. If the pip install step produced an error for it:

```bash
# macOS: point at Homebrew's libpq
export PATH="$(brew --prefix postgresql)/bin:$PATH"
pip install psycopg2 --no-cache-dir
```

Quick smoke test:
```bash
python -c "import psycopg2; print('psycopg2 OK:', psycopg2.__version__)"
```

## Step 6 — PostgreSQL role

Odoo needs a dedicated PostgreSQL role with `CREATEDB` privilege (it refuses to run as
`postgres` for security). The dashboard creates this automatically, but if you're
running manually:

```bash
# Check if the role already exists
psql -tAc "SELECT 1 FROM pg_roles WHERE rolname='odoo'" postgres

# Create it if missing
psql -c "CREATE ROLE odoo LOGIN PASSWORD 'odoo' CREATEDB;" postgres
```

The `odoo.conf` generated by the dashboard uses `db_user = odoo` / `db_password = odoo`.
These match the role above.

> **Note:** On macOS with Homebrew PostgreSQL, the default superuser is your OS username.
> If `psql postgres` fails, try `psql -U $(whoami) postgres` or check that the
> PostgreSQL service is running: `brew services start postgresql@16`.

## Step 7 — wkhtmltopdf

Required for PDF report generation. **Not a pip package** — must be a system binary.

| Odoo version | wkhtmltopdf version |
|---|---|
| 14.0, 15.0 | 0.12.5 (patched Qt) |
| 16.0 – 19.0 | 0.12.6 (patched Qt) |

**macOS (Intel or Apple Silicon via Rosetta):**
Download the `.pkg` from https://github.com/wkhtmltopdf/packaging/releases
and install manually. The Homebrew `wkhtmltopdf` formula **does not** include the
patched Qt build needed for headers/footers.

Verify:
```bash
wkhtmltopdf --version
```

> For demo purposes wkhtmltopdf is optional — Odoo runs fine without it, PDF reports
> just won't render. Skip if not needed.

## Step 9 — Smoke-test odoo-bin

```bash
ODOO_SRC="odoo-workspace/<version>/odoo"
python "$ODOO_SRC/odoo-bin" --version
```

Expected output: `Odoo Server <version>` (e.g., `Odoo Server 17.0`).

If this prints the version without errors, the source install is complete.

## Troubleshooting

**`lxml` build fails on Python 3.12+:**
```bash
pip install lxml-html-clean   # separate package split from lxml 5.x
```

**`gevent` / `greenlet` build fails:**
These need a C compiler. On macOS ensure Xcode tools are installed.
For Python 3.13 specifically, use `gevent>=24.11.1` and `greenlet>=3.1.1`.

**`xlrd 2.x` can't read `.xls` files (v18/v19):**
`xlrd 2.0+` dropped `.xls` support. If importing old Excel files, install `openpyxl`
(already in requirements for v18/v19) and convert to `.xlsx` first.

**ImportError for `odoo` module:**
This is normal — `odoo-bin` bootstraps itself. Always run via `python odoo-bin`, never
`import odoo` directly.
