## Key Fields for CSV Loading

| Field | Type | Required | Default | Notes |

| `team_id` | Many2one (quality.alert.team) | Yes | First team found | FK resolved by name. Raises UserError if no team exists |

| `test_type_id` | Many2one (quality.point.test_type) | Yes | First test type | Available types: `passfail` (Pass-Fail), `measure` (Measure), `instructions` (Instructions), `picture` (Take a Picture), `spreadsheet` (Spreadsheet) |
| `company_id` | Many2one (res.company) | Yes | current company | Company |

| `measure_frequency_type` | Selection | Yes | `'all'` | `all`, `random`, `periodical`, `on_demand`. Added by quality_control |

## Constraints

- No SQL constraints on this model.
- `show_failure_location` is computed: failure locations hidden for `instructions` and `picture` test types.

## create() Override

- If `name` is not provided or equals `'New'`, auto-generates via `ir.sequence` with code `quality.point` (prefix `QCP`, padding 5).
- No other field-level side effects.

## display_name Computation (quality_control)

```python
display_name = f'{point.name} - {point.title}' if point.title else point.name
```

So a record with `name='QCP00001'` and `title='Visual Inspection'` displays as "QCP00001 - Visual Inspection".

## Odoo Demo Data Patterns

From `quality_control/data/quality_control_demo.xml`:
```xml
<record id="quality_point1" model="quality.point">
    <field name="product_ids" eval="[(4, ref('product.product_product_27'))]"/>
    <field name="picking_type_ids" eval="[(4, ref('stock.picking_type_in'))]"/>
    <field name="note">Quality Control For Drawer's Parts...</field>
</record>
```

Key patterns:
- `name` not set (auto-generated by sequence)
- `title` not set in demo data
- `team_id` not set (uses default -- first team)
- `test_type_id` not set (uses default -- first type, typically `passfail`)
- `product_ids` links via M2M command `(4, id)` to specific products
- `picking_type_ids` links to stock operation types (e.g. Receipts)
- `note` provides instructions text

## CSV Loading Notes

- **Identity key**: `title` is the best human-readable key. If `title` is empty, `name` (the auto-generated sequence) can be used, but it is less stable.
- **name field**: OMIT from CSV or set to `New` -- the sequence auto-generates it. Do NOT invent QCP numbers.
- **picking_type_ids**: M2M field. In CSV, provide comma-separated operation type names (e.g., "Receipts,Delivery Orders"). The loader resolves via `name_search` on `stock.picking.type`.
- **product_ids**: M2M field. Provide comma-separated product names.
- **test_type_id**: Use the test type **name** (e.g., "Pass - Fail", "Measure", "Instructions", "Take a Picture"), not the technical name.
- **team_id**: Resolve by name (e.g., "Main Quality Team"). A team MUST exist or create() raises UserError.
- **Load order**: Must load AFTER `quality.alert.team`, `product.product`, `stock.picking.type`.
- **Measurement fields**: Only set `norm`, `tolerance_min`, `tolerance_max`, `norm_unit` when `test_type_id` refers to the "Measure" type.

["title"]
